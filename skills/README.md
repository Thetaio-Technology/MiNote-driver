# minote-skill

`minote-skill` 把 `minote-driver` 的底层自动化能力，包装成更适合 agent、产品集成方和终端体验层使用的 skill 能力。

它的目标不是再造一个浏览器驱动，而是把已经验证过的执行能力，变成一句话能理解、一个入口能调用、一个结果能返回的上层能力层。

## 它解决什么问题

如果只有 `driver`，你拿到的是：

- Selenium
- CLI
- 选择器
- 登录态复用
- 工程接口

如果有了 `minote-skill`，你拿到的是：

- 清晰的能力边界
- 统一的调用入口
- 面向 agent 的输入输出约定
- 更接近真实产品体验的能力封装

换句话说：

- `minote-driver` 负责把事做成
- `minote-skill` 负责把这件事变得更容易被调用

## 当前能力

当前第一个能力包是：

- `minote-todo`

它已经能完成：

- 读取未完成待办
- 读取已完成待办
- 创建待办
- 修改待办标题
- 标记完成
- 恢复待办
- 删除待办

## 适合谁

适合：

- 想把小米云笔记待办接进 agent 的开发者
- 想做个人工作流自动化的人
- 想把待办操作封装成产品能力的人
- 想要一个本地、可控、真实可跑的执行后端的人

## 使用体验目标

`minote-skill` 想提供的不是“再写一层说明文档”，而是更像下面这种体验：

- 你告诉上层系统要做什么
- 上层把意图映射成结构化命令
- skill 调 `minote-driver` 执行
- 返回稳定结果给用户或 agent

典型入口：

```bash
python scripts/cli/run_skill.py minote-todo create --title "明天下午买咖啡豆"
```

## 设计原则

- 不做中文自然语言解析器
- 优先复用已验证过的执行能力
- 尽量把 skill 保持为薄封装
- 优先统一输入输出，而不是新增复杂抽象
- 保持本地可控和可验证

## 当前目录内容

推荐的 skill 包内容：

- `SKILL.md`：能力定义、边界、运行规则
- `interface.md`：输入输出协议
- `examples.md`：典型调用示例
- `checklist.md`：执行前后检查项

## 下一步适合怎么发展

如果后续拆成独立仓库，`minote-skill` 会更适合作为：

- 对外展示的 skill 仓库
- agent 平台接入层
- 机器可读 manifest 的承载层
- 多个 MiNote 能力包的集合入口
